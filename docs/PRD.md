# JobFit Agent PRD

## 1. 서비스 목적

JobFit Agent는 취업준비생이 목표 기업의 채용공고와 자신의 경험을 입력하면, 채용공고의 담당업무, 필수사항, 우대사항, 인재상과 사용자 경험 사이의 차이를 분석하고 실행 가능한 준비 방향을 제안하는 AI Agent 서비스다.

서비스의 목적은 단순한 채용공고 요약이 아니다. 사용자가 지금 가진 기술스택, 프로젝트 경험, 자기소개서 또는 경험 서술을 기준으로 어떤 역량을 보완해야 하는지, 어떤 프로젝트를 만들거나 개선해야 하는지, 남은 기간 동안 어떤 순서로 준비해야 하는지를 한국어 Markdown 리포트로 제공하는 것이다.

## 2. 핵심 사용자

- 신입 개발자 또는 주니어 개발자로 취업을 준비하는 사용자
- 부트캠프, 국비교육, 대학교 프로젝트 경험을 취업 포트폴리오로 정리하려는 사용자
- 특정 기업 또는 직무 공고에 맞춰 자기소개서와 프로젝트 경험을 조정하려는 사용자
- 제한된 준비 기간 안에서 학습 우선순위와 프로젝트 방향을 정해야 하는 사용자
- 채용공고의 요구사항을 읽어도 실제 준비 항목으로 바꾸기 어려운 사용자

## 3. 사용자가 겪는 문제

사용자는 채용공고를 읽어도 담당업무, 필수사항, 우대사항이 자신의 경험과 어떻게 연결되는지 판단하기 어렵다.

주요 문제는 다음과 같다.

- 채용공고 문장을 실제 역량 요구사항으로 분해하기 어렵다.
- 본인의 프로젝트 경험이 어떤 요구사항을 증명하는지 알기 어렵다.
- 부족한 역량이 무엇인지 알아도 공부 순서를 정하기 어렵다.
- 준비 기간이 짧을수록 무엇을 포기하고 무엇을 집중해야 하는지 판단하기 어렵다.
- 자기소개서나 경험 서술이 공고와 인재상에 맞는지 객관적으로 보기 어렵다.
- 프로젝트를 새로 해야 하는지, 기존 프로젝트를 보완하면 되는지 판단하기 어렵다.

## 4. 기존 취업 상담 방식의 한계

기존 취업 상담은 사람의 경험과 조언에 기반한다는 장점이 있지만, 다음 한계가 있다.

- 상담자의 전문 분야와 경험에 따라 조언 품질이 크게 달라진다.
- 특정 채용공고의 문장을 세밀하게 분해해 사용자 경험과 매핑하기 어렵다.
- 상담 시간이 제한되어 반복적인 자기소개서, 프로젝트, 로드맵 수정이 어렵다.
- 사용자가 공고를 바꿀 때마다 새로 상담을 받아야 한다.
- 상담 결과가 정형 데이터로 남지 않아 비교, 수정, 리포트 생성이 어렵다.
- 학습 로드맵과 프로젝트 추천이 사용자의 남은 기간에 맞춰 충분히 구체화되지 않는 경우가 많다.

## 5. JobFit Agent가 해결할 문제

JobFit Agent는 채용공고와 사용자 경험을 같은 기준으로 구조화해 비교한다.

해결하려는 문제는 다음과 같다.

- 채용공고의 담당업무, 필수사항, 우대사항, 인재상을 역량 단위로 분해한다.
- 사용자 기술스택, 프로젝트 경험, 자기소개서 또는 경험 서술을 역량 단위로 정리한다.
- 공고 요구사항과 사용자 경험 사이의 일치, 부족, 증거 부족 영역을 구분한다.
- 부족한 역량을 준비 기간 기준으로 우선순위화한다.
- 사용자가 실행 가능한 학습 로드맵과 프로젝트 추천을 제공한다.
- 결과를 Markdown 리포트로 생성해 복사, 저장, 검토하기 쉽게 만든다.

## 6. MVP 범위

MVP는 로그인과 데이터베이스 없이 동작하는 단일 세션 분석 서비스다. 사용자는 텍스트를 수동 입력하고, 서버는 OpenAI API를 호출해 구조화된 분석 결과를 생성한 뒤 Markdown 리포트를 만든다.

MVP에 포함한다.

- 채용공고 텍스트 수동 입력
- 회사 인재상 텍스트 수동 입력
- 사용자 기술스택 입력
- 사용자 프로젝트 경험 입력
- 자기소개서 또는 경험 서술 입력
- AI 기반 공고 분석
- AI 기반 사용자 역량 분석
- 역량 갭 분석
- 학습 로드맵 추천
- 프로젝트 추천
- 프로젝트 난이도 제안
- 실행 로드맵 제안
- Markdown 리포트 생성
- 한국어 UI 제공

MVP 성공 기준은 다음과 같다.

- 사용자는 하나의 분석 흐름에서 입력부터 리포트 확인까지 완료할 수 있다.
- API Key는 서버에서만 사용되고 클라이언트에 노출되지 않는다.
- 사용자가 입력한 자기소개서와 개인정보는 저장되지 않는다.
- AI 출력은 JSON Schema 기반 구조화 출력을 우선하고, 그 결과를 Markdown 리포트로 변환한다.
- 결과는 합격 가능성을 단정하지 않고 준비 방향과 실행 과제를 제안한다.

## 7. MVP에서 제외할 기능

MVP에서는 다음 기능을 제외한다.

- 로그인
- DB 저장
- 분석 히스토리 저장
- 실제 채용공고 크롤링
- 결제
- GitHub 자동 분석
- 이력서 PDF 자동 파싱
- 벡터 DB 기반 본격 RAG
- 회사별 공고 자동 수집
- 파일 업로드
- 다국어 UI
- 관리자 페이지
- 이메일 발송
- 합격 가능성 점수화 또는 보장

## 8. 주요 화면 구성

### 8.1 홈 화면

서비스 목적과 사용 흐름을 짧게 보여준다. 사용자는 분석 시작 버튼으로 입력 화면에 진입한다.

### 8.2 입력 화면

사용자가 분석에 필요한 정보를 입력하는 화면이다.

입력 영역은 다음과 같다.

- 목표 기업
- 목표 직무
- 채용공고 텍스트
- 회사 인재상 텍스트
- 사용자 기술스택
- 사용자 프로젝트 경험
- 자기소개서 또는 경험 서술
- 준비 가능 기간
- 하루 학습 가능 시간 또는 제약 조건

입력 화면에는 개인정보 저장 안 함, 민감정보 입력 주의, AI 결과 검토 필요 안내를 표시한다.

### 8.3 분석 진행 화면

분석 요청 후 진행 상태를 표시한다. 중복 제출을 막기 위해 요청 처리 중에는 제출 버튼을 비활성화한다.

### 8.4 결과 화면

구조화된 분석 결과와 Markdown 리포트를 표시한다.

주요 섹션은 다음과 같다.

- 채용공고 분석
- 사용자 역량 분석
- 역량 갭 분석
- 학습 로드맵
- 프로젝트 추천
- 프로젝트 난이도
- 실행 로드맵
- 자기소개서 또는 경험 서술 개선 포인트
- 최종 Markdown 리포트

### 8.5 오류 화면

필수 입력 누락, API 실패, 구조화 출력 실패, 네트워크 실패를 한국어로 안내한다. 오류가 발생해도 사용자 입력 원문은 저장하지 않는다.

## 9. 주요 사용자 흐름

### 9.1 기본 분석 흐름

1. 사용자가 홈 화면에서 분석 시작을 선택한다.
2. 사용자가 채용공고, 인재상, 기술스택, 프로젝트 경험, 자기소개서 또는 경험 서술을 입력한다.
3. 사용자가 준비 기간과 학습 가능 시간을 입력한다.
4. 클라이언트가 서버 API로 입력값을 전송한다.
5. 서버가 필수 입력과 길이 제한을 검증한다.
6. 서버가 OpenAI API를 호출해 공고와 사용자 경험을 분석한다.
7. 서버가 JSON Schema 기반 구조화 응답을 검증한다.
8. 서버가 구조화 응답을 Markdown 리포트로 변환한다.
9. 클라이언트가 결과 화면에 분석 결과와 Markdown 리포트를 표시한다.
10. 사용자는 AI 결과를 검토하고 필요한 부분을 직접 수정해 활용한다.

### 9.2 입력 보완 흐름

1. 사용자가 필수 입력을 누락한 상태로 제출한다.
2. 시스템이 누락된 항목을 표시한다.
3. 사용자가 입력을 보완한다.
4. 사용자가 다시 분석을 요청한다.

### 9.3 결과 검토 흐름

1. 사용자가 결과 화면에서 AI가 제안한 역량 갭과 프로젝트 추천을 확인한다.
2. 사용자가 실제 경험과 다른 내용이 있는지 검토한다.
3. 사용자가 과장되었거나 사실과 다른 문장을 제거한다.
4. 사용자가 Markdown 리포트를 자기소개서 수정, 프로젝트 계획, 학습 계획에 활용한다.

## 10. 입력 데이터

MVP 입력 데이터는 다음과 같다.

| 항목 | 필수 여부 | 설명 | 예시 |
| --- | --- | --- | --- |
| 목표 기업 | 필수 | 지원하려는 회사명 | 네이버, 카카오, 토스 |
| 목표 직무 | 필수 | 지원하려는 직무 | 백엔드 개발자, AI 서비스 개발자 |
| 채용공고 텍스트 | 필수 | 담당업무, 필수사항, 우대사항이 포함된 공고 본문 | 공고 복사 텍스트 |
| 회사 인재상 텍스트 | 선택 | 회사의 핵심 가치, 문화, 인재상 | 고객 중심, 주도성, 협업 |
| 사용자 기술스택 | 필수 | 사용자가 사용할 수 있는 기술 | Python, FastAPI, React, SQL |
| 사용자 프로젝트 경험 | 필수 | 프로젝트명, 역할, 사용 기술, 결과 | AI 챗봇, 대시보드, 임베디드 프로젝트 |
| 자기소개서 또는 경험 서술 | 선택 | 지원동기, 문제 해결 경험, 성장 경험 | 자기소개서 초안 |
| 준비 가능 기간 | 필수 | 지원 전까지 남은 기간 | 2주, 1개월, 3개월 |
| 학습 가능 시간 | 선택 | 하루 또는 주 단위 학습 가능 시간 | 평일 2시간, 주말 6시간 |
| 제약 조건 | 선택 | 선호 기술, 피해야 할 범위, 프로젝트 조건 | 백엔드 중심, 배포 포함 |

민감정보 입력은 요구하지 않는다. 주민등록번호, 주소, 전화번호, 계좌번호 같은 개인정보는 입력하지 않도록 안내한다.

## 11. 출력 데이터

AI 출력은 JSON Schema 기반 구조화 출력을 우선한다. Markdown 리포트는 검증된 구조화 결과를 사람이 읽기 좋은 문서로 변환한 산출물이다.

MVP 출력 데이터는 다음과 같다.

| 항목 | 설명 |
| --- | --- |
| 공고 분석 | 담당업무, 필수사항, 우대사항, 인재상 요약 |
| 요구 역량 | 기술 역량, 협업 역량, 문제 해결 역량, 도메인 역량 |
| 사용자 역량 분석 | 기술스택과 프로젝트 경험에서 확인되는 강점 |
| 역량 갭 | 부족한 역량, 증거가 약한 역량, 우대사항 대비 차이 |
| 우선순위 | 준비 기간과 난이도를 고려한 보완 순서 |
| 학습 로드맵 | 기간별 학습 주제와 실행 과제 |
| 프로젝트 추천 | 기존 프로젝트 보완안 또는 신규 프로젝트 아이디어 |
| 프로젝트 난이도 | 쉬움, 보통, 어려움 등 MVP 수준 난이도 |
| 실행 로드맵 | 프로젝트 진행 단계, 산출물, 검증 방법 |
| 자기소개서 개선 포인트 | 공고와 인재상에 맞춘 표현 개선 방향 |
| 주의사항 | AI 분석 한계, 사실 검증 필요, 과장 표현 주의 |
| Markdown 리포트 | 위 결과를 하나의 한국어 문서로 정리한 결과 |

## 12. OpenAI API 사용 지점

OpenAI API Key는 서버 환경 변수로만 관리한다. 클라이언트 코드, 브라우저 번들, HTML, 로그에 API Key를 포함하지 않는다.

MVP에서 OpenAI API를 사용하는 지점은 다음과 같다.

- 채용공고의 담당업무, 필수사항, 우대사항 추출
- 회사 인재상과 공고 요구사항의 연결 분석
- 사용자 기술스택과 프로젝트 경험 분석
- 자기소개서 또는 경험 서술의 공고 적합성 분석
- 공고 요구사항과 사용자 경험 사이의 역량 갭 분석
- 학습 로드맵 생성
- 프로젝트 추천과 난이도 제안
- Markdown 리포트 생성을 위한 구조화 데이터 생성

OpenAI API 사용 원칙은 다음과 같다.

- 클라이언트는 서버 API에만 요청한다.
- 서버만 OpenAI API를 호출한다.
- 서버 전용 코드는 클라이언트에서 import하지 않는다.
- 모든 AI 출력은 가능한 한 JSON Schema 기반 구조화 출력으로 요청한다.
- 서버는 AI 응답 구조를 검증한 뒤 클라이언트에 전달한다.
- API 실패 시 원문 자기소개서와 개인정보를 저장하지 않는다.

## 13. 개인정보 보호 원칙

MVP 개인정보 보호 원칙은 다음과 같다.

- API Key는 서버에서만 사용한다.
- 클라이언트에 API Key를 노출하지 않는다.
- 로그인과 데이터베이스를 사용하지 않는다.
- 사용자가 입력한 자기소개서와 개인정보는 저장하지 않는다.
- 서버 로그에 자기소개서 원문, 전화번호, 이메일, 주소 등 개인정보를 남기지 않는다.
- 분석 완료 후 요청 데이터는 재사용하지 않는다.
- 사용자가 민감정보를 입력하지 않도록 입력 화면에서 안내한다.
- AI 결과는 참고용이며 최종 제출 전 사용자가 직접 검토해야 함을 안내한다.

## 14. Human-in-the-loop 설계

JobFit Agent는 AI가 최종 판단자가 아니라 사용자의 검토를 돕는 분석 도구로 설계한다.

MVP의 Human-in-the-loop 원칙은 다음과 같다.

- 사용자가 입력한 경험이 사실의 기준이다.
- AI는 공고와 경험 사이의 연결 가능성을 제안하지만 사실 여부를 확정하지 않는다.
- 결과 화면에서 사용자가 직접 검토해야 할 항목을 명확히 표시한다.
- 자기소개서 개선 포인트는 바로 제출 가능한 문장보다 수정 방향 중심으로 제공한다.
- 프로젝트 추천은 사용자가 실제 수행 가능 여부를 판단할 수 있도록 난이도와 실행 단계를 함께 제공한다.
- Markdown 리포트에는 "검토 필요", "경험 근거 부족", "과장 주의" 같은 주의 표시를 포함한다.
- 사용자는 AI 결과를 그대로 제출하지 않고 자신의 실제 경험에 맞게 수정해야 한다.

## 15. 실패 시나리오

| 시나리오 | 원인 | 사용자 안내 | 시스템 처리 |
| --- | --- | --- | --- |
| 필수 입력 누락 | 공고, 기술스택, 프로젝트 경험 등 미입력 | 누락 항목 입력 요청 | API 호출 전 차단 |
| 입력 길이 초과 | 공고나 자기소개서가 너무 김 | 핵심 내용만 줄여서 입력 요청 | API 호출 전 차단 |
| 공고 내용 부족 | 담당업무나 요구사항이 불명확함 | 더 구체적인 공고 입력 요청 | 낮은 신뢰도 안내 |
| OpenAI API 실패 | 네트워크, 한도, 서버 오류 | 잠시 후 재시도 안내 | 입력 데이터 저장 안 함 |
| 구조화 출력 실패 | AI 응답이 JSON Schema와 불일치 | 다시 시도 안내 | 응답 폐기 후 오류 반환 |
| 개인정보 포함 | 사용자가 민감정보 입력 | 민감정보 제거 권장 | 저장하지 않음 |
| 과장된 결과 | AI가 입력 근거보다 강하게 표현함 | 직접 검토 및 수정 안내 | 주의사항 표시 |
| 비현실적 로드맵 | 준비 기간 대비 요구사항이 많음 | 핵심 우선순위 중심으로 축소 안내 | 필수 액션 중심 출력 |
| 클라이언트 네트워크 실패 | 사용자 네트워크 문제 | 연결 확인 후 재시도 안내 | 서버 저장 없음 |

## 16. 향후 확장 기능

MVP 검증 이후 다음 기능을 고려한다.

- 채용공고 URL 자동 분석
- 실제 채용공고 크롤링

- 이력서 PDF 자동 파싱
- 벡터 DB 기반 RAG
- 직무별 역량 템플릿
- 회사별 인재상 템플릿
- 자기소개서 버전 관리
- 프로젝트 README 개선 제안
- 모의 면접 질문 생성
- 주간 학습 체크리스트
- 캘린더 기반 학습 일정 생성
- 분석 결과 PDF 내보내기
- 교육기관 또는 멘토용 대시보드

## 구현 전제

- 기본 UI 언어는 한국어다.
- MVP는 저장 없는 단일 세션 분석 서비스다.
- 서버는 OpenAI API 호출과 응답 검증을 담당한다.
- 클라이언트는 사용자 입력과 결과 표시를 담당한다.
- Markdown 리포트는 구조화된 분석 결과에서 생성한다.
- 보안과 개인정보 보호는 MVP에서도 제외하지 않는다.
