# JobFit Agent Knowledge Base

## 백엔드 개발 직무 역량
백엔드 개발자는 API 설계, 데이터 모델링, 입력 검증, 인증/인가, 테스트, 배포, 장애 대응 경험을 요구받는 경우가 많다. 포트폴리오는 단순 CRUD보다 요구사항을 API와 데이터 모델로 바꾸고, 실패 케이스와 운영 로그를 설명할 수 있어야 한다.

## IT 인프라 직무 역량
IT 인프라 직무는 Linux 운영, 네트워크 기본기, Docker 또는 Nginx 배포, 모니터링, 로그 분석, 장애 대응 런북 작성 능력이 중요하다. 산출물은 점검 스크립트, 장애 재현 기록, 모니터링 화면, 운영 README가 좋다.

## 임베디드 소프트웨어 직무 역량
임베디드 소프트웨어 직무는 C/C++, MCU, GPIO, UART, CAN, FreeRTOS, 상태 머신, 시리얼 로그 기반 디버깅 역량을 본다. 포트폴리오는 핀맵, 재현 절차, 로그, 테스트 결과를 함께 남겨야 설득력이 높다.

## 프로젝트 추천 원칙
추천 프로젝트는 기존 경험과 연결되어야 하며, 난이도는 준비 기간과 현재 수준에 맞춰야 한다. 입문은 1~2주 단일 기능, 중급은 3~6주 API/저장/문서화, 고급은 6~12주 배포/테스트/운영까지 포함하는 범위가 적절하다.

## 학습 로드맵 원칙
학습 로드맵은 공부 항목만 나열하면 안 된다. 각 주차마다 학습 주제, 실습 과제, 프로젝트 작업, 포트폴리오 산출물이 연결되어야 한다. 완료 기준은 README, 테스트 결과, 실행 캡처, 회고 문서처럼 확인 가능한 결과물이어야 한다.
