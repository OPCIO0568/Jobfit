# IT 인프라 직무 RAG 지식 문서

이 문서는 JobFit Agent의 로컬 RAG 검색 대상이다. 실제 기업 공고가 아니라 발표와 테스트를 위한 일반화된 IT 인프라 직무 기준이다.

## 1. 직무별 핵심 역량

- Linux 서버 운영과 기본 명령어 활용
- 네트워크 기본 구조 이해: TCP/IP, DNS, HTTP, 방화벽, 포트
- 배포 환경 구성: Docker, Nginx, reverse proxy, SSL 기본
- 로그 확인과 장애 원인 추적
- 모니터링 지표를 보고 문제를 설명하는 능력
- 운영 절차를 문서화하고 반복 가능한 형태로 만드는 능력

## 2. 필수 기술

- Linux shell, systemd, 권한 관리, 프로세스 확인
- 네트워크 진단 도구: ping, curl, netstat 또는 ss, traceroute
- Docker 컨테이너 실행, 이미지 빌드, 볼륨과 환경변수 관리
- Nginx 기본 설정, 정적 파일 서빙, 프록시 설정
- 로그 수집과 분석: application log, access log, error log
- 간단한 Bash 또는 Python 운영 스크립트

## 3. 우대 기술

- Prometheus, Grafana, Loki 같은 모니터링 도구 경험
- CI/CD를 통한 배포 자동화
- 클라우드 기본 지식: IAM, VPC, security group, load balancer
- 장애 대응 런북 작성 경험
- 백업, 복구, 권한 분리, 보안 점검 경험

## 4. 입문/중급/고급 수준 기준

- 입문: Linux 명령어로 서버 상태를 확인하고 Docker로 단일 서비스를 실행할 수 있다.
- 중급: Nginx 프록시, 로그 분석, 배포 스크립트, 장애 재현 문서를 만들 수 있다.
- 고급: 모니터링 대시보드, 알림, 장애 대응 런북, 무중단에 가까운 배포 흐름을 설계할 수 있다.

## 5. 추천 프로젝트 예시

- 개인 서버 운영 점검 대시보드: CPU, 메모리, 디스크, 포트 상태를 수집하고 화면 또는 Markdown 리포트로 출력한다.
- Docker + Nginx 배포 실습: 웹 앱을 컨테이너로 실행하고 Nginx reverse proxy와 HTTPS 적용 과정을 문서화한다.
- 장애 대응 런북 프로젝트: 의도적으로 장애를 만들고 탐지, 원인 분석, 복구, 재발 방지 절차를 기록한다.

## 6. 포트폴리오 산출물

- 시스템 구성도와 포트 흐름도
- Dockerfile, compose 파일, Nginx 설정 파일
- 장애 재현 로그와 복구 절차
- 모니터링 화면 캡처 또는 지표 예시
- 운영 README와 체크리스트

## 7. 면접에서 검증될 수 있는 질문

- 서버가 느려졌을 때 어떤 순서로 확인하는가?
- 502 Bad Gateway가 발생하면 무엇을 점검하는가?
- Docker 컨테이너가 재시작되는 원인을 어떻게 찾는가?
- Nginx access log와 error log를 어떻게 다르게 활용하는가?
- 배포 실패 시 롤백 절차를 어떻게 설계했는가?

## 8. 주의사항

- 단순히 도구 이름만 나열하면 운영 역량으로 보기 어렵다.
- 장애 대응 프로젝트는 반드시 재현 조건, 로그, 복구 절차가 있어야 한다.
- 클라우드 비용이 발생할 수 있으므로 데모는 로컬 또는 무료 범위에서 설계한다.
- 보안 토큰, 서버 비밀번호, 개인 IP 등 민감정보를 포트폴리오에 노출하지 않는다.
