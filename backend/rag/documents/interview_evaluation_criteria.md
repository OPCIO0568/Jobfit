# 면접 평가 기준 RAG 지식 문서

이 문서는 JobFit Agent의 로컬 RAG 검색 대상이다. 프로젝트와 학습 로드맵이 면접 검증 관점에서 충분한지 판단하는 일반 기준을 제공한다.

## 1. 직무별 핵심 역량

- IT 인프라: 장애를 발견하고 복구하며 재발 방지 절차를 설명하는 능력
- 백엔드 개발: 요구사항을 안정적인 API와 데이터 모델로 구현하는 능력
- 임베디드 소프트웨어: 제한된 하드웨어 환경에서 동작을 검증하고 문제를 분리하는 능력
- 공통: 모르는 것을 인정하고 근거 기반으로 설명하는 태도

## 2. 필수 기술

- 문제를 작게 나누는 능력
- 로그, 테스트, 문서 같은 객관적 근거 제시
- 기술 선택 이유 설명
- 실패 원인과 개선 방향 정리
- 보안과 개인정보 보호 기본 인식

## 3. 우대 기술

- 운영 또는 배포 경험
- 협업 기록과 코드 리뷰 경험
- 성능 개선 또는 장애 대응 사례
- 반복 가능한 테스트 자동화
- 사용자 관점의 문제 정의

## 4. 입문/중급/고급 수준 기준

- 입문: 구현한 기능과 사용한 기술을 정확히 설명할 수 있다.
- 중급: 설계 이유, 실패 케이스, 테스트 방법을 함께 설명할 수 있다.
- 고급: 운영 이슈, 성능, 확장성, 보안, 협업 과정까지 근거로 설명할 수 있다.

## 5. 추천 프로젝트 예시

- 면접 질문 역추적 프로젝트: 목표 공고의 요구역량별 예상 질문과 답변 근거를 정리한다.
- 기술 선택 회고 프로젝트: 사용한 라이브러리, 프레임워크, 데이터 구조 선택 이유를 문서화한다.
- 실패 사례 분석 프로젝트: 오류 로그, 원인, 해결, 재발 방지 대책을 하나의 문서로 만든다.

## 6. 포트폴리오 산출물

- 예상 질문과 답변 근거 표
- 기술 선택 이유 문서
- 테스트와 실패 케이스 정리
- 프로젝트 회고
- 본인 기여도와 역할 구분

## 7. 면접에서 검증될 수 있는 질문

- 이 프로젝트에서 본인이 직접 한 일은 무엇인가?
- 어떤 기준으로 기술을 선택했는가?
- 가장 큰 실패는 무엇이었고 어떻게 해결했는가?
- 테스트하지 못한 부분은 무엇인가?
- 공고 요구역량 중 아직 부족한 부분은 무엇이라고 보는가?

## 8. 주의사항

- 취업 성공을 보장하는 표현을 사용하지 않는다.
- 프로젝트 결과보다 판단 근거와 학습 과정을 명확히 남긴다.
- 모르는 내용을 아는 것처럼 말하지 않는다.
- 공고에 없는 역량을 필수 요건처럼 부풀리지 않는다.
