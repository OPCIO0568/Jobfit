# JobFit Agent 데모 시나리오

## 1. 데모 목적

3~5분 안에 JobFit Agent가 채용공고와 사용자 경험을 비교해 역량 갭, 추천 프로젝트, 학습 로드맵, Markdown 리포트까지 생성하는 흐름을 보여준다.

데모에서 강조할 목표는 다음과 같다.

- 채용공고를 단순 요약하지 않고 요구역량으로 구조화한다.
- 사용자의 실제 프로젝트 경험과 공고 요구사항 사이의 차이를 보여준다.
- AI 추천을 그대로 확정하지 않고 사용자가 검토하고 수정한다.
- 최종 결과를 Markdown 파일로 저장할 수 있다.
- `MOCK_AI=true`로 API Key 없이도 발표가 가능하다.

## 2. 데모 사용자 페르소나

- 이름: 발표용 가상 사용자
- 상태: 백엔드 개발자 신입 취업 준비생
- 보유 경험: TypeScript, Node.js, PostgreSQL, Zod 기반 API 프로젝트
- 고민: 공고에서 요구하는 인증, 테스트, 배포, 운영 경험을 어떻게 포트폴리오로 증명해야 할지 모름
- 준비 기간: 8주
- 선호 프로젝트 방식: 개인 프로젝트
- 현재 수준: 중급

## 3. 입력할 채용공고 예시

데모에서는 샘플 데이터의 `백엔드 개발 직무` 버튼을 사용한다.

핵심 채용공고 내용:

```text
담당업무
- REST API 설계 및 구현
- 데이터 모델링과 SQL 쿼리 성능 개선
- 인증/인가, 입력 검증, 에러 처리 정책 구현
- 테스트 코드와 API 문서 작성
- 배포 환경에서 로그를 확인하고 문제를 추적

필수사항
- Node.js 또는 Python 기반 API 서버 개발 경험
- 관계형 데이터베이스 스키마 설계 경험
- HTTP, 인증, 트랜잭션, 예외 처리 기본 이해
- Git 기반 협업과 코드 리뷰 경험

우대사항
- TypeScript, NestJS, FastAPI, PostgreSQL 경험
- Docker 기반 개발 환경 구성
- CI/CD, 테스트 자동화, 모니터링 경험
```

회사 인재상 예시:

```text
사용자 문제를 API와 데이터 모델로 명확히 풀어내는 개발자를 선호합니다.
빠르게 만드는 것뿐 아니라 입력 검증, 장애 대응, 문서화를 함께 고려하는 태도를 중요하게 봅니다.
불확실한 요구사항은 질문으로 좁히고, 결정 내용을 기록해 팀과 공유하는 문화를 지향합니다.
```

## 4. 입력할 사용자 경험 예시

샘플 데이터의 사용자 경험 요약:

```text
기술스택
TypeScript, Node.js, Express, PostgreSQL, Prisma, Docker, Jest, GitHub Actions

프로젝트 경험
채용 준비 리포트 API 프로젝트
- 사용자의 입력을 Zod로 검증하고, 분석 결과를 JSON 형태로 반환하는 API를 구현했습니다.
- PostgreSQL 스키마를 설계하고 Prisma로 CRUD와 목록 조회를 구성했습니다.
- 에러 응답 형식을 통일하고 README에 API 요청/응답 예시를 정리했습니다.
- 결과물: API 명세, DB schema, 테스트 케이스, 배포 URL

경험 서술
백엔드 프로젝트에서 단순히 데이터를 저장하는 것보다 입력 검증과 에러 응답 형식이 중요하다는 점을 경험했습니다.
처음에는 기능 구현에 집중했지만, 테스트 케이스와 API 문서를 추가하면서 다른 사람이 사용할 수 있는 형태로 개선했습니다.
앞으로는 성능 측정, 인증/인가, 배포 후 로그 기반 문제 추적까지 보완하고 싶습니다.
```

## 5. 분석 진행 흐름

발표 전 준비:

```bash
npm run dev
```

`.env.local`은 발표 안정성을 위해 다음처럼 둔다.

```env
OPENAI_API_KEY=
OPENAI_MODEL=
MOCK_AI=true
```

데모 순서:

1. 브라우저에서 `http://localhost:3000`을 연다.
2. 상단의 `MOCK_AI ON` 표시를 보여준다.
3. 샘플 데이터 버튼에서 `백엔드 개발 직무`를 선택한다.
4. Wizard 단계별로 입력값이 채워진 것을 빠르게 확인한다.
5. 마지막 단계에서 `분석 시작`을 누른다.
6. 진행 상태를 보며 다음 단계가 순서대로 실행된다고 설명한다.
   - 채용공고 분석 중
   - 사용자 경험 분석 중
   - 역량 갭 분석 중
   - 프로젝트 추천 생성 중
   - 학습 로드맵 생성 중
   - 최종 리포트 생성 대기

## 6. 결과 화면에서 강조할 포인트

결과 화면에서는 다음 순서로 짧게 보여준다.

1. 채용공고 핵심 요구역량
   - 공고 문장이 기술, 협업, 문서화, 도메인 역량으로 구조화된다.
2. 사용자 보유 역량
   - 사용자가 실제로 입력한 프로젝트 경험에서 확인되는 역량만 보여준다.
3. 부족 역량
   - 운영, 배포, 모니터링, 인증/인가처럼 근거가 약한 영역을 확인한다.
4. 공부로 보완할 항목
   - 학습 주제가 프로젝트 산출물과 연결된다.
5. 프로젝트로 증명할 항목
   - 단순 공부가 아니라 README, 테스트 결과, 배포 URL 같은 증거를 남기도록 유도한다.
6. 추천 프로젝트 3개
   - 단기, 중급 포트폴리오, 고급 확장 프로젝트로 난이도가 나뉜다.
7. 4주/8주/12주 로드맵
   - 준비 기간이 달라도 주차별 산출물을 확인할 수 있다.

## 7. Human-in-the-loop 장면

AI 추천을 바로 확정하지 않는 장면을 보여준다.

시연 순서:

1. Human-in-the-loop 패널로 이동한다.
2. 기준 프로젝트를 하나 선택한다.
3. 피드백에 다음 문장을 입력한다.

```text
이 추천은 너무 어려움. 8주 안에 완성 가능한 실무형 백엔드 포트폴리오로 줄여줘.
```

4. `선택 프로젝트 기준으로 로드맵 재생성`을 누른다.
5. 로드맵이 사용자 피드백 기준으로 다시 생성된다고 설명한다.
6. `사용자 승인 후 최종 리포트 생성`을 누른다.

강조 멘트:

```text
이 서비스는 AI가 최종 결정을 대신하지 않습니다.
사용자가 준비 기간, 난이도, 프로젝트 방향을 검토한 뒤 승인해야 최종 리포트가 생성됩니다.
```

## 8. Markdown 리포트 다운로드 장면

최종 리포트 생성 후 다음을 보여준다.

1. Markdown 미리보기에서 전체 리포트 구조를 확인한다.
2. 개인정보가 포함될 수 있으므로 다운로드 전 경고 문구를 보여준다.
3. `클립보드 복사` 버튼을 누를 수 있다고 설명한다.
4. `.md` 다운로드 버튼으로 `jobfit-report-YYYYMMDD.md` 파일을 저장한다.

강조할 리포트 구성:

- 목표 직무 요약
- 채용공고 요구역량
- 사용자 보유 역량
- 역량 갭 분석
- 공부해야 할 내용
- 추천 프로젝트
- 학습 로드맵
- 포트폴리오 산출물 체크리스트
- 주의사항
- AI 분석 한계

## 9. 실패 시나리오

발표 중 시간이 남으면 실패 처리도 짧게 보여준다.

### 9.1 입력값이 너무 짧은 경우

1. 채용공고 원문을 `짧은 공고`처럼 짧게 입력한다.
2. 분석을 실행한다.
3. 시스템이 추가 입력 요청을 표시한다.

보여줄 메시지 방향:

```text
채용공고 내용이 부족합니다.
담당업무, 필수사항, 우대사항이 드러나도록 공고 내용을 더 붙여넣어 주세요.
```

### 9.2 사용자 경험이 부족한 경우

1. 프로젝트 경험을 `프로젝트 했음`처럼 짧게 입력한다.
2. 분석을 실행한다.
3. 분석 한계와 보완 요청을 표시한다.

보여줄 메시지 방향:

```text
사용자 경험 내용이 부족해 분석 한계가 큽니다.
프로젝트에서 맡은 역할, 구현한 기능, 사용 기술, 결과물을 더 입력해 주세요.
```

### 9.3 API Key 누락 시나리오

실제 API 모드에서만 설명한다. 발표 중에는 직접 전환하지 않아도 된다.

```text
MOCK_AI=false인데 OPENAI_API_KEY가 없으면 서버는 내부 스택을 보여주지 않고 설정 안내 메시지를 반환합니다.
API Key는 서버 환경변수에서만 읽고 클라이언트로 전달하지 않습니다.
```

## 10. 발표자가 말할 핵심 멘트

### 시작 멘트

```text
JobFit Agent는 채용공고와 사용자 경험을 같은 구조로 분석해서,
내가 무엇을 이미 증명했고 무엇을 더 준비해야 하는지 보여주는 취업 준비 AI Agent입니다.
```

### 입력 화면 멘트

```text
MVP에서는 크롤링이나 로그인 없이 텍스트를 직접 입력합니다.
이렇게 단순하게 만든 이유는 채용공고 기반 분석 흐름을 먼저 검증하기 위해서입니다.
```

### 분석 진행 멘트

```text
분석은 공고 분석, 사용자 경험 분석, 갭 분석, 프로젝트 추천, 로드맵 생성 순서로 실행됩니다.
각 단계의 AI 출력은 Zod Schema로 검증해서 화면이 제멋대로 깨지지 않도록 했습니다.
```

### 결과 화면 멘트

```text
중요한 점은 합격 가능성을 점수로 단정하지 않는다는 것입니다.
대신 공고 근거, 사용자 경험 근거, 부족한 증거를 나눠서 보여줍니다.
```

### Human-in-the-loop 멘트

```text
AI 추천은 초안입니다.
사용자가 난이도와 기간을 조정하고 승인해야 최종 리포트가 생성됩니다.
```

### 개인정보 보호 멘트

```text
MVP에서는 로그인과 DB 저장을 사용하지 않습니다.
API Key는 서버에서만 사용하고, 사용자가 입력한 자기소개서나 개인정보는 저장하지 않습니다.
```

### 마무리 멘트

```text
이 프로젝트의 핵심은 취업 상담을 대체하는 것이 아니라,
채용공고를 실행 가능한 학습 계획과 포트폴리오 과제로 바꾸는 보조 도구를 만드는 것입니다.
```
