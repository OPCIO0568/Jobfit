# 프로젝트 추천 템플릿 RAG 지식 문서

이 문서는 JobFit Agent의 로컬 RAG 검색 대상이다. 직무별 추천 프로젝트를 만들 때 난이도, 기간, 산출물 기준을 일반화해 제공한다.

## 1. 직무별 핵심 역량

- IT 인프라: 운영 자동화, 배포, 로그 분석, 모니터링, 장애 대응
- 백엔드 개발: API 설계, 데이터 모델링, 검증, 테스트, 배포
- 임베디드 소프트웨어: MCU 제어, 통신, 상태 머신, 디버깅 로그, 테스트 절차

## 2. 필수 기술

- 문제 정의와 요구사항 정리
- 최소 동작 버전 구현
- 실행 방법 문서화
- 테스트 또는 검증 결과 기록
- 포트폴리오에서 설명 가능한 산출물 정리

## 3. 우대 기술

- 배포 또는 실제 실행 환경 구성
- 자동화 스크립트
- 모니터링, 로그, 오류 처리
- 성능 또는 안정성 개선 기록
- 팀 협업 방식: issue, branch, pull request, 회고

## 4. 입문/중급/고급 수준 기준

- 입문: 1~2주, 단일 핵심 기능, 로컬 실행, README와 실행 캡처 포함
- 중급: 3~6주, API/저장/시각화/문서화 중 2개 이상 포함
- 고급: 6~12주, 배포/운영/테스트/모니터링/협업 흐름 포함

## 5. 추천 프로젝트 예시

- 입문 프로젝트: 채용공고 키워드 추출기, 서버 상태 점검 스크립트, UART 로그 뷰어
- 중급 프로젝트: 포트폴리오 분석 API, Docker 배포 대시보드, CAN 로그 분석 도구
- 고급 프로젝트: JobFit Agent 백엔드, 장애 대응 훈련 플랫폼, RTOS 기반 진단 모니터

## 6. 포트폴리오 산출물

- 문제 정의와 대상 사용자
- 주요 기능 목록과 제외 범위
- 아키텍처 다이어그램
- 실행 방법과 환경변수 설명
- 테스트 결과와 한계
- 개선 전후 비교 또는 회고

## 7. 면접에서 검증될 수 있는 질문

- 이 프로젝트가 목표 직무의 어떤 역량을 증명하는가?
- 왜 이 범위를 MVP로 정했는가?
- 가장 어려웠던 문제와 해결 근거는 무엇인가?
- 시간이 더 있다면 무엇을 개선하겠는가?
- 단순 기능 구현을 넘어 운영 또는 검증을 어떻게 고려했는가?

## 8. 주의사항

- 프로젝트 추천은 사용자의 실제 경험과 준비 기간을 기준으로 축소 가능해야 한다.
- 너무 큰 프로젝트는 축소 버전과 완료 기준을 함께 제시해야 한다.
- 공고에 없는 요구사항을 필수처럼 단정하지 않는다.
- 사용자가 수행하지 않은 경험을 이미 수행한 것처럼 표현하지 않는다.
